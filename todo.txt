 Phase 0：基础修复（类型安全 + 环境配置）

 0.1 修复 Bonus 追踪机制

 问题： bonus 触发标志（_y_c0、_b_b2 等）用下划线前缀存在 player 对象上，不在 Schema
 中，断线后丢失。

 修复方案：
 - 在 server/schema/GameState.ts 的 PlayerState 中新增 earnedBonuses: ArraySchema<string> 字段
 - 记录已获得 bonus 的标识（如 "y_c0", "b_b2"）
 - 替换 CleverRoom.ts 中所有 (player as any)._xxx 读写为 schema 字段操作

 0.2 修复 pendingDieId 类型安全

 问题： (client as any).pendingDieId 不类型安全

 修复方案：
 - 在 PlayerState schema 中增加 pendingDieId: string 字段，或在 CleverRoom 用 Map<string, string>
 维护 sessionId → pendingDieId 的映射

 0.3 环境变量配置

 修复：
 - 客户端新建 client/.env.local (开发) 和 client/.env.production
 - 添加 NEXT_PUBLIC_SERVER_URL=ws://localhost:2567
 - 修改 client/app/page.tsx:7 的硬编码地址为 process.env.NEXT_PUBLIC_SERVER_URL
 - 服务端 server/index.ts 已支持 process.env.PORT，确认 CORS 也使用环境变量配置

 Phase 1：游戏逻辑完善

 1.1 完成被动玩家阶段（CleverRoom.ts:101 TODO）

 当前问题： 被动玩家点"完成"直接跳过，不做任何标记。

 完整实现：
 - finishPassiveTurn handler 接收 { dieId: string | null }
   - null 表示跳过（没有合适的格子可标记）
   - 非 null 时：查找 silverPlatter 中的 die，调用与主动玩家相同的 markDie 逻辑
 - 在 PlayerState 中增加 passiveDone: boolean schema 字段替代 _passive_done hack
 - 所有被动玩家标记完成（或跳过）后，进入下一轮

 1.2 断线重连支持

 实现方案（使用 Colyseus 内置 allowReconnection）：

 在 CleverRoom.ts 的 onLeave 中：
 async onLeave(client: Client, consented: boolean) {
   // 标记玩家为离线状态（不删除）
   const player = this.state.players.get(client.sessionId);
   if (player) {
     player.isConnected = false;
   }

   // 等待最多 60 秒重连
   try {
     await this.allowReconnection(client, 60);
     // 重连成功
     player.isConnected = true;
   } catch {
     // 超时，真正离开
     this.state.players.delete(client.sessionId);
     this.advanceTurnIfNeeded();  // 如果是当前活跃玩家，跳过其回合
   }
 }

 Schema 新增字段：
 - PlayerState.isConnected: boolean
 - PlayerState.passiveDone: boolean
 - PlayerState.earnedBonuses: ArraySchema<string>

 客户端重连逻辑：
 - 检测 room.onError 和连接断开事件
 - 尝试 client.reconnect(roomId, sessionId) 重新加入
 - 显示"正在重连..."UI 状态

 Phase 2：完整房间系统

 2.1 服务端房间 API

 在 server/index.ts 新增 REST 端点：

 GET  /api/rooms          - 列出所有等待中的房间
 POST /api/rooms          - 创建新房间（可设置房间名、最大人数）
 GET  /api/rooms/:roomId  - 获取特定房间信息

 房间元数据通过 Colyseus matchMaker.query() 获取，结合房间自定义 metadata。

 2.2 房间 Schema 增强

 在 GameStateSchema 增加：
 - roomName: string - 房间名（玩家自定义）
 - hostSessionId: string - 房主 session
 - createdAt: number - 创建时间戳

 2.3 房间创建选项

 修改 CleverRoom onCreate(options) 支持：
 {
   roomName: string,    // 自定义房间名
   maxRounds?: number,  // 4（默认）
   maxPlayers?: number, // 2-6
 }

 2.4 客户端房间大厅页面

 新文件 client/app/lobby/page.tsx：
 - 显示当前所有开放房间列表（房间名、人数、状态）
 - "创建新房间"按钮（弹窗输入房间名）
 - "通过房间码加入"输入框
 - 每 3 秒自动刷新房间列表

 Phase 3：对局记录系统

 3.1 SQLite 数据库设置

 新建 server/db/database.ts，使用 better-sqlite3：

 CREATE TABLE games (
   id TEXT PRIMARY KEY,         -- 游戏唯一ID（UUID）
   room_id TEXT,                -- Colyseus 房间ID
   room_name TEXT,
   started_at INTEGER,          -- Unix timestamp
   ended_at INTEGER,
   total_rounds INTEGER,
   winner_name TEXT
 );

 CREATE TABLE game_players (
   id INTEGER PRIMARY KEY AUTOINCREMENT,
   game_id TEXT REFERENCES games(id),
   player_name TEXT,
   total_score INTEGER,
   yellow_score INTEGER,
   blue_score INTEGER,
   green_score INTEGER,
   orange_score INTEGER,
   purple_score INTEGER,
   fox_bonus INTEGER,
   rerolls_used INTEGER,
   rank INTEGER                 -- 最终名次
 );

 3.2 游戏结束时写入

 Phase 5：部署配置（Docker）

 5.1 服务端 Dockerfile

 server/Dockerfile：
 FROM node:20-alpine
 WORKDIR /app
 COPY package*.json ./
 COPY server/package*.json ./server/
 COPY shared/package*.json ./shared/
 RUN npm ci --workspace=server --workspace=shared
 COPY server/ ./server/
 COPY shared/ ./shared/
 EXPOSE 2567
 CMD ["npm", "run", "start", "--workspace=server"]

 5.2 客户端 Dockerfile

 client/Dockerfile（Next.js standalone 模式）：
 FROM node:20-alpine AS builder
 WORKDIR /app
 COPY . .
 RUN npm ci && npm run build --workspace=client
 FROM node:20-alpine
 COPY --from=builder /app/client/.next/standalone ./
 EXPOSE 3000
 CMD ["node", "server.js"]

 5.3 docker-compose.yml

 services:
   server:
     build: { context: ., dockerfile: server/Dockerfile }
     ports: ["2567:2567"]
     environment:
       - PORT=2567
       - CORS_ORIGIN=http://your-domain.com
       - DB_PATH=/data/clever.db
     volumes:
       - db_data:/data

   client:
     build: { context: ., dockerfile: client/Dockerfile }
     ports: ["3000:3000"]
     environment:
       - NEXT_PUBLIC_SERVER_URL=ws://your-domain.com:2567
     depends_on: [server]

 volumes:
   db_data:

 5.4 Nginx 反向代理配置（可选）

 nginx.conf：
 - / → Next.js client:3000
 - /ws → Colyseus server:2567（WebSocket 代理）
 - /api → Colyseus server:2567（REST API）

 统一 80/443 端口，避免暴露内部端口。

 验证方法

 1. 游戏逻辑： 本地启动两个浏览器窗口，玩一局完整游戏（4轮）验证被动阶段、bonus 触发
 2. 断线重连： 游戏进行中关闭客户端标签页，60秒内重新打开，验证重连恢复状态
 3. 多桌并发： 创建两个房间，各自进行游戏，验证互不干扰
 4. 对局记录： 完成一局后访问 /history 页面，验证记录保存
 5. Docker 部署： docker-compose up 后从浏览器访问，验证所有功能正常
