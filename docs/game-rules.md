# Game Rules Breakdown

最后更新：`2026-03-30`

这个文档用于把纸面规则拆成可实现的程序规则。当前版本先把最关键的“主动回合三次选骰 + 银盘 + 被动选骰”精确化，区域录入和奖励链细则继续后补。

## 1. 规则来源

- 主规则书：`Ganz_schon_clever.pdf`
- 计分纸参考：`prettyclever_w_scoring_black.pdf`
- 在线交叉核对：

## 2. 当前已经锁定的事实

- 每位主动玩家最多进行三次选骰
- 每次选中一个骰子后，所有点数更小的骰子进入银盘
- 银盘中的骰子不能再被主动玩家继续使用
- 如果主动玩家在第一或第二次选择后，已经没有骰子可重掷，那么本回合会提前结束
- 主动玩家完成最多三次选择后，所有剩余未使用骰子都进入银盘
- 只有在主动玩家完成主动部分后，其他玩家才进入被动选择阶段
- 每个被动玩家最多从银盘选择一个骰子
- 被动玩家是同时从银盘中选择，因此多个被动玩家可以选择同一个银盘骰子
- 一轮结束的定义是：每位玩家都做过一次主动玩家

## 3. 当前实现范围

本轮只实现下面这些规则：

- 开局创建第一位主动玩家
- 主动玩家掷骰
- 主动玩家选骰
- 更小点数骰子进入银盘
- 剩余骰子是否可继续重掷
- 提前结束主动阶段
- 被动玩家依次提交选择结果
- 回合结束后推进到下一位主动玩家
- 玩家级别的每回合临时状态重置

本轮暂不实现：

- 五个颜色区域的录入合法性
- 白色万能骰的跨区域特殊用法
- 奖励链与额外动作
- 分区计分
- 终局触发条件

## 4. 回合状态机

### `awaiting_active_roll`

- 当前主动玩家应提交本次可重掷骰子的结果
- 只有当前仍在手里的骰子可以被掷出

### `awaiting_active_selection`

- 主动玩家从当前掷出的骰子中选择一个
- 选中的骰子记入主动选择列表
- 点数更小的骰子进入银盘
- 点数更大或相等且未被选中的骰子，进入下一次重掷候选

### `awaiting_passive_picks`

- 主动阶段结束后进入
- 非主动玩家每人最多选择一个银盘骰子
- 被动玩家选择不会消耗银盘，其他被动玩家仍可选同一个骰子

### `awaiting_turn_end`

- 主动与被动部分都结束
- 等待推进到下一位主动玩家

## 5. 主动回合算法

1. 主动玩家拥有当前仍可使用的骰子集合
2. 提交一次掷骰结果
3. 从结果中选出一个骰子
4. 所有更小点数骰子加入银盘
5. 如果还未选满三次且仍有骰子可重掷，则回到 `awaiting_active_roll`
6. 否则，主动阶段结束
7. 若还有其他玩家，进入 `awaiting_passive_picks`
8. 若没有其他玩家，直接进入 `awaiting_turn_end`

## 6. 当前数字化解释

- 银盘是“共享可见但不被被动玩家消耗”的资源池
- 被动选择阶段在服务端上可以按顺序收集，但语义上等价于同时选择
- 规则引擎只接受“动作意图”，不直接接受前端改状态
