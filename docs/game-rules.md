# Game Rules Breakdown

最后更新：`2026-03-31`

这个文档用于把纸面规则拆成可实现的程序规则。当前版本先把最关键的“主动回合三次选骰 + 银盘 + 被动选骰”精确化，区域录入和奖励链细则继续后补。

## 1. 规则来源

- 主规则书：`Ganz_schon_clever.pdf`
- 计分纸参考：`prettyclever_w_scoring_black.pdf`
- 在线交叉核对：
  - [Tesera 英文规则译本](https://tesera.ru/images/items/1269156/Ganz_schon_clever.pdf)
  - [UltraBoardGames 规则页](https://www.ultraboardgames.com/thats-pretty-clever/game-rules.php)

## 2. 当前已经锁定的事实

- 每位主动玩家最多进行三次选骰
- 每次选中一个骰子后，所有点数更小的骰子进入银盘
- 银盘中的骰子不能再被主动玩家继续使用
- 如果主动玩家在第一或第二次选择后，已经没有骰子可重掷，那么本回合会提前结束
- 主动玩家完成最多三次选择后，所有剩余未使用骰子都进入银盘
- 只有在主动玩家完成主动部分后，其他玩家才进入被动选择阶段
- 每个被动玩家最多从银盘选择一个骰子
- 被动玩家是同时从银盘中选择，因此多个被动玩家可以选择同一个银盘骰子
- 一轮结束的定义是：每位玩家都做过一次主动玩家

## 3. 当前实现范围

本轮只实现下面这些规则：

- 开局创建第一位主动玩家
- 主动玩家掷骰
- 主动玩家选骰
- 更小点数骰子进入银盘
- 剩余骰子是否可继续重掷
- 提前结束主动阶段
- 被动玩家依次提交选择结果
- 回合结束后推进到下一位主动玩家
- 玩家级别的每回合临时状态重置

本轮暂不实现：

- 五个颜色区域的录入合法性
- 白色万能骰的跨区域特殊用法
- 奖励链与额外动作
- 分区计分
- 终局触发条件

## 4. 回合状态机

### `awaiting_active_roll`

- 当前主动玩家应提交本次可重掷骰子的结果
- 只有当前仍在手里的骰子可以被掷出

### `awaiting_active_selection`

- 主动玩家从当前掷出的骰子中选择一个
- 选中的骰子记入主动选择列表
- 点数更小的骰子进入银盘
- 点数更大或相等且未被选中的骰子，进入下一次重掷候选

### `awaiting_passive_picks`

- 主动阶段结束后进入
- 非主动玩家每人最多选择一个银盘骰子
- 被动玩家选择不会消耗银盘，其他被动玩家仍可选同一个骰子

### `awaiting_turn_end`

- 主动与被动部分都结束
- 等待推进到下一位主动玩家

## 5. 主动回合算法

1. 主动玩家拥有当前仍可使用的骰子集合
2. 提交一次掷骰结果
3. 从结果中选出一个骰子
4. 所有更小点数骰子加入银盘
5. 如果还未选满三次且仍有骰子可重掷，则回到 `awaiting_active_roll`
6. 否则，主动阶段结束
7. 若还有其他玩家，进入 `awaiting_passive_picks`
8. 若没有其他玩家，直接进入 `awaiting_turn_end`

## 6. 当前数字化解释

- 银盘是“共享可见但不被被动玩家消耗”的资源池
- 被动选择阶段在服务端上可以按顺序收集，但语义上等价于同时选择
- 规则引擎只接受“动作意图”，不直接接受前端改状态

## 7. 仍待逐条核对的部分

- yellow / blue 显式格子的 printed 常量与奖励位，仍需继续对照原始 score sheet 逐格复核
- 白骰在所有区域中的边界交互，仍建议继续用真人长局核对
- “立即数字奖励 / 万能填写”在完全无合法落点时的丢失规则，当前数字版还没有统一做成显式跳过动作
- Solo 模式仍未实现

## 8. 当前修复计划

1. 继续对照原始 score sheet 复核 yellow / blue 的 printed 常量、奖励位和 tooltip 文案。
2. 给“无法执行的即时数字奖励 / 万能填写”补统一的失效或跳过处理，避免回合中出现奖励解析卡住。
3. 做更多 2-4 人真人长局，重点盯白骰跨区、bonus chain、终局尾声协同。
4. 视首发范围决定是否继续补 Solo 模式。

## 9. 当前实现备注

- `green / orange / purple` 的落点合法性已经在 `game-core` 中落地
- 绿区当前按“填入数字 `>=` 图纸上该格门槛数字即可”的解释实现
- `green / orange / purple` 的线性奖励位现在会按当前数字版常量触发 bonus：
  - 已接线：`X`、`FOX`、`+1`、定值 `6`
  - `R` 现在会累计为“重投动作”，并允许主动玩家在已掷骰但还没选骰时重掷当前可用骰组
- `yellow / blue` 已从简化计数升级成显式格子模型：
  - `yellow` 当前实现为带 `4 x 4` 视觉布局的显式格子模型，其中包含 12 个可填写数字格和 4 个预印 `X` 占位；规则层仍保留 3 个奖励轨与 1 个对角奖励
  - `blue` 当前实现为显式和数格子、按行和按列触发奖励
- bonus queue / resolve 已接入黄蓝奖励触发
- 即时奖励现在会累计到玩家资源区：`extra-die`、`fox`、`reroll`
- 当前数字版对资源消费的解释：
  - `reroll`：仅允许主动玩家在 `awaiting_active_selection` 阶段消费，效果是重掷这一次仍可用的整组骰子
  - `extra-die`：已改成真实的“回合末额外再选 1 颗骰子”；当前实现允许主动玩家在主动部分结束后、被动玩家在自己的常规被动选择结束后消费
  - `extra-die` 当前会从本回合全部 6 颗骰子中选目标，并记录“同一玩家本回合已用 extra-die 选过哪些骰子”，防止同一颗骰子被这个动作重复选择
- 前四轮 round tracker 已接入：
  - 第 1 轮：所有玩家开局获得 `1` 个 `reroll`
  - 第 2 轮：所有玩家开局获得 `1` 个 `extra-die`
  - 第 3 轮：所有玩家开局再获得 `1` 个 `reroll`
  - 第 4 轮：所有玩家依次在黑色 `X` 与黑色 `6` 之间二选一，并立即执行
- 当主动阶段当前所有已掷骰子都没有任何合法落点时，当前实现会把这次掷骰记为空过，并消耗一次常规掷骰机会；若这已经是最后一次常规掷骰，则主动部分结束
- 当被动玩家无法使用任何银盘骰子时，当前实现会要求其改从主动玩家的 3 个已选骰位里拿 1 颗；只有银盘和主动骰位都没有合法落点时，才允许跳过
- 4 人局当前按 `4` 轮结算；`3` 人局 `5` 轮；`1-2` 人局 `6` 轮
- 终局尾声当前按纸规解释：
  - 最后一位主动玩家完成后，若仍有人可使用 `extra-die`，必须继续使用或明确放弃后，回合才能结束
  - 未使用的 `reroll` 会在最终结算前作废
  - standings 平分时先比“单一区域最高分”，再不分则共享名次
- 区域计分、总分汇总、终局 standings 已接入 `game-core`
- 已补自动化回归覆盖：`extra-die`、主动空过、被动主动骰位补救、round tracker、第 4 轮二选一、4 人轮数、终局 `reroll / extra-die / tie-break`
- 黄蓝常量当前属于“高可信原型”，仍待按原始 score sheet 逐格复核
